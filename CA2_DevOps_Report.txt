1. PROJECT SYNOPSIS (10 Marks)
-----------------------------------------
Project Title: Automated CI/CD Pipeline for MKPDesigns (MERN Stack)
Tools Used: Docker, Jenkins, GitHub Actions
Architecture/Workflow:
- Code Push -> GitHub Repository
- Pull Request -> GitHub Actions (CI: Linting & Testing)
- Merge to Main -> Webhook triggers Jenkins
- Jenkins (CD) -> Build Docker Images -> Push to DockerHub -> Deploy Containers
Expected Results: Automated, zero-downtime deployments with consistent Dockerized environments.

A. Dockerization

Execution Details (Phase 3A):
- Implemented Microservices Architecture by creating isolated `Dockerfile`s for `client` and `server`.
- Configured `.dockerignore` files to prevent unnecessary files (like `node_modules` and `.env`) from being copied into the images, ensuring security and optimizing build sizes.
- Created a `docker-compose.yml` to orchestrate the containers, mapping port 5000 for the backend and 5173 for the frontend, while securely injecting environment variables from local `.env` files.
- Committed the containerization logic to a feature branch (`feature/dockerize-app`) and successfully merged into `main`.

B. GitHub Actions (Continuous Integration)
[Screenshot Slot: GitHub Actions workflow file ci.yml]
[Screenshot Slot: GitHub Actions interface showing a successful build/test run (Green ticks)]

Execution Details (Phase 3B):
- Established a Continuous Integration pipeline by creating `.github/workflows/ci.yml`.
- Implemented DevSecOps practices by adding a `security.yml` action to perform automated dependency vulnerability scanning using `npm audit`.
- Added a `docker-check.yml` action to automatically dry-run Docker builds on Pull Requests, preventing broken infrastructure code from reaching the `main` branch.
- Configured the workflow to trigger automatically on `push` and `pull_request` events to the `main` branch.
- Automated the setup of the Node.js environment, installation of dependencies for both `client` and `server`, and compilation of the client build to ensure code integrity.
- Committed the CI workflow via a dedicated feature branch (`feature/github-actions` and `feature/additional-actions`) and merged into `main` to trigger the first run.
